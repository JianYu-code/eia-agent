## 对比结论摘要

### 参考平台审核产物结构（94 页 PDF 实测）
1. 交叉检查问题（核心模块）：跨表数值一致性、表内加和校验、批次×单批=年产量重算、前后章数据矛盾（投资额 48万 vs 48.8万）、Q值表 vs 原辅料表、标准限值按排气筒高度内插（GB16297 20m/30m→22m）、经纬度度分秒误填十进制、单位错误（mg/L vs mg/cm³）、表格编号重复/标题与内容不符、监测时间不一致

3. 工程分析及污染防治问题（追加）／环境影响预测及风险章节问题（追加）／固废、危废交叉检查（追加）
4. 图文不一致检查
5. 法规、标准名称及编号专项检查
6. 逐章问题（按章节组织）
7. 每条意见带「最小化修改指引（供参考）」+「意见来源：【XX类】审核意见案例汇总」
8. 首页使用说明：明示总体采纳率 40+%（最高 70%），教用户快速概览、优先看追加模块
9. 工作流：上传→付费门槛→管理员复核→批准/拒绝→下载审核意见（状态：待审核/已审核/已批准/已拒绝）

- P0/P1/P2 分级 + A/B/C/D 评级 + Top3 + 专家组总结（参考平台无分级无评级）
- 每条意见带推理链 reasoning + law_ref + evidence_location 原文高亮
- 步骤三态徽章（pass/fail/error），异常不静默
- 反馈闭环落库反哺 prompt + 管理端准确率/误报率统计
- 三域分管线（环评 11 步 / 验收 7 阶段 / 应急 5 步）+ 资料包完整性
- K 文件注入 + 章节路由 + per-chapter map-reduce；视觉模型图文分型检查
- 标准废止精确判定 + 替代关系；队列 + 真停止 + 并发限速；Word/PDF 导出

## O1 — 交叉数据核验引擎（核心，分三期实施）

### O1a 表格结构化提取 + 确定性校验（无 LLM，高置信零误报成本）
- `extractor.py` 增加表格抽取：
  - DOCX：`python-docx` 遍历 `document.tables`，提取 caption（表前/后一段「表X.Y-Z」）、headers、rows，记录所属章节
  - PDF：PyMuPDF `page.find_tables()`，扫描件跳过（交由视觉通道）
  - 输出 `text_data["tables"] = [{caption, number, headers, rows, chapter}]`；`full_text` 现有逻辑不受影响
- 新建 `engine/steps/crosscheck.py`（纯确定性，全确定性 issue 直接标高置信）：
  - **表内加和校验**：识别含「合计/总计/小计」行或列 → 数值列分项累加 vs 合计值，相对偏差 >1%（或绝对差 > 末位精度）→ issue（rule_id R-XCHK-SUM）
  - **表格编号重复/断号**：全文正则提取「表X.Y-Z」编号序列 → 重复编号、编号断档 → issue（R-XCHK-TBN）
  - **经纬度误填**：度分秒坐标与十进制坐标同文出现时，检测「分/秒数值 ≥60」被当作小数位填入（如 126.27567961° vs 126°27′58″）→ 两坐标换算后偏差 >1km → issue（R-XCHK-GEO）
  - **表头与列数不符/列定义不清**：rows 列数不一致 → P2 提示（R-XCHK-TBL）
- 接入 `audit_runner.EIA_STEPS` 新增步骤「12 交叉数据核验」（acceptance 域同样启用，emergency 域 skip）
- issue 字段规范：evidence_location = 表号 + 单元格原文片段；chapter 由表格归属直接给出

### O1b 跨表/跨章同一指标一致性（确定性候选 + LLM 仲裁，控误报）
- **指标抽取**：正则 + 单位词典从全文/表格抽取「指标实体」：投资额、环保投资、新水用量、排水量、用电量、蒸汽量、各污染物排放量、危废产生量、最大储存量、Q 值、监测日期等 → `{name, value, unit, location(chapter/表号), raw}`
- **候选配对**：同名（或别名表：投资额↔总投资、排水量↔废水量）+ 单位可换算 → 配对；数值相对偏差 >5%（日期类不等）→ 候选矛盾对
- **LLM 仲裁**：候选对批量送 LLM 一次调用判定「是否同一指标口径、是否真矛盾」→ 真矛盾输出 issue（R-XCHK-CONS，含两处定位、两值、差值、最小化修改建议）
- 典型案例覆盖（来自参考平台实测）：投资额 48万 vs 48.8万；物料平衡表水量 vs 水平衡表；Q值表三氯化铝 50.526t vs 原辅料表 60t；「无环境问题」结论 vs 以新带老修复裂痕（结论-证据矛盾，LLM 判定）

### O1c 重算类核验（扩展 recalculate 思路）
- **批次×单批=年产量**：物料平衡/产污表中同行「单批产量 × 年批次数」vs「年产量」列 → 偏差 >2% → issue（R-XCHK-BATCH，确定性）
- **标准限值内插**：排气筒高度 h 落在标准两档 h1/h2 之间 → 线性内插限值 vs 报告引用值，偏差 >10% → issue（R-XCHK-INT；内置 GB16297 等常用标准的速率-高度档表，优先走 standards_index/KB 取档值）
- **速率↔总量换算**：kg/h × 年运行小时 vs t/a 列 → 偏差 >5% → issue（确定性，年运行时间从文中提取）
- **单位量级**：同指标同单位在不同表出现量级差 10^n 倍 → 候选 → LLM 仲裁
- 与现有 `recalculate.py`（锅炉/废水系数反算）并存，统一归入「12 交叉数据核验」步骤

### 验证（O1）
- 构造含已知错误的测试报告（加和错、跨章投资额不一致、批次错、22m 内插错、经纬度误填、表号重复）→ 全部检出且定位到表号
- 参考平台 94 页 PDF 中交叉检查模块前 20 条问题类型 → 本项目同类型覆盖率 ≥80%
- 无误报基线：3 份正确报告 → 确定性检查 0 误报；LLM 仲裁候选驳回率 >50% 视为正常过滤
- 单测：表格提取（DOCX/PDF）、加和判定容差、坐标换算、内插计算、候选配对归一化

## O3 — 最小化修改指引
- `run_llm_check`、`run_llm_check_per_chapter`、`crosscheck` LLM 仲裁的 prompt 统一要求 suggestion 字段输出「最小化修改指引」格式：明确指出把哪段原文的什么数值/表述改成什么（例：「将年产量 755.63 t/a 修正为 775.61 t/a，并同步更新表 10.2-3」）
- 报告模板 suggestion 标签改为「最小化修改指引（供参考）」（对齐参考平台措辞）
- 验证：抽样 10 条 LLM issue，suggestion 含具体数值/原文片段比例 ≥70%

## O4 — 术语与敏感词规范检查（低优先，可配置）
